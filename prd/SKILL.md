# PRD 转开发文档 Skill

## 触发条件

- 手动触发: 输入 `/prd`、`产品需求`、`生成技术文档`、`生成文档`
- 自动触发: 当用户提供需求文档、产品说明、业务需求时

## 概述

这是一个将产品需求文档（PRD）转换为AI开发用技术文档的Skill。支持读取Markdown格式的PRD文件或直接输入需求描述，输出完整的开发技术文档。

---

## 输入要求

### 方式一：读取PRD文件

提供PRD文件的路径，Skill会自动读取并解析：

```
请根据这个PRD生成技术文档: C:/path/to/prd.md
```

### 方式二：直接输入需求

直接描述产品需求，越详细越好：

```
做一个用户登录注册功能...
```

---

## 输出：完整开发文档

生成的文档包含以下章节：

### 1. 功能概述

- 需求背景
- 功能目标
- 用户场景
- 核心业务流程图（文字描述）

### 2. 技术选型

| 组件 | 选型建议 | 理由 |
|-----|---------|-----|
| 后端框架 | 根据项目现有技术栈 | 保持一致性 |
| 数据库 | 根据数据量和事务需求 | 性能与开发效率平衡 |
| 缓存 | 根据访问模式 | 热点数据加速 |
| 消息队列 | 根据异步需求 | 解耦与削峰 |

### 3. 数据库设计

**ER关系说明**

列出核心实体及其关系。

**表结构设计**

```
表名: xxx
字段:
  - id: 主键
  - xxx: 字段说明
  ...
索引:
  - idx_xxx: 索引字段
```

### 4. API接口设计

**接口列表**

| 接口 | 方法 | 路径 | 说明 |
|-----|------|-----|------|
| xxx | POST | /api/xxx | 功能说明 |

**详细接口定义**

请求参数、响应格式、错误码说明。

### 5. 模块设计

**核心类/接口**

- XxxService: 业务逻辑
- XxxController: 请求处理
- XxxDAO: 数据访问

**关键方法**

```
方法名: processXxx
输入: XXX
输出: XXX
逻辑: 描述
```

### 6. 业务流程

用文字描述核心业务流程和分支处理。

### 7. 安全考虑

- 权限控制
- 参数校验
- 敏感数据处理
- 日志脱敏

### 8. 性能考虑

- 缓存策略
- 异步处理
- 索引优化

### 9. 测试策略

- 单元测试覆盖点
- 集成测试场景
- 关键边界条件

---

## 使用方法

1. **提供PRD**: 直接粘贴需求文本，或提供.md文件路径
2. **指定技术栈**（可选）: 如有特定技术要求可以说明
3. **获取文档**: Skill自动生成完整技术文档

示例：

```
/prd
需求：做一个积分商城，用户可以用积分兑换商品
- 用户积分余额查询
- 商品列表展示（可用积分兑换）
- 兑换记录
```

或者：

```
根据这个PRD生成技术文档: D:/docs/prd.md
```
